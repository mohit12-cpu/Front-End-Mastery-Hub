1. Introduction to Git

Git is a distributed version control system that tracks changes in any set of computer files, usually used for coordinating work among programmers collaboratively developing source code during software development.

What is Version Control?

Version control is a system that records changes to files over time so that you can recall specific versions later.

  • Backup: Keep track of all changes to your code
  • Collaboration: Work with others on the same project
  • History: See who made what changes and when
  • Branching: Work on different features simultaneously
  • Merging: Combine different versions of your code

Key Concepts

# Repository (Repo)
# A Git repository contains all of your project's files and history

# Commit
# A snapshot of your project at a specific point in time

# Branch
# A parallel version of your repository

# Merge
# Combining changes from different branches

2. Installation & Setup

Installing Git and setting up your environment is the first step to using version control.

Installing Git

Windows: Download Git from git-scm.com

Mac: Install via Homebrew: brew install git

Linux: Use your package manager: sudo apt install git

Initial Configuration

# Set your identity
git config --global user.name "John Doe"
git config --global user.email "john@example.com"

# Set your default editor
git config --global core.editor "code --wait"

# Set default branch name
git config --global init.defaultBranch main

# Enable color output
git config --global color.ui auto

Verifying Installation

# Check Git version
git --version

# Check configuration
git config --global user.name
git config --global user.email

# View all settings
git config --list

3. Basic Commands

Learn the essential Git commands to start tracking your projects.

Creating a Repository

# Initialize a new Git repository
git init

# Clone an existing repository
git clone https://github.com/user/repo.git

Tracking Changes

# Check status of files
git status

# Add files to staging area
git add filename.txt
git add .  # Add all changes

# Commit changes
git commit -m "Add new feature"

# View commit history
git log

Working with Remotes

# Add a remote repository
git remote add origin https://github.com/user/repo.git

# Push changes to remote
git push origin main

# Pull changes from remote
git pull origin main

# View remote repositories
git remote -v

4. Branching and Merging

Branching allows you to work on different features or experiments without affecting the main codebase.

Working with Branches

# View all branches
git branch

# Create a new branch
git branch feature-login

# Switch to a branch
git checkout feature-login

# Create and switch to a new branch (shortcut)
git checkout -b feature-profile

# Create and switch with newer syntax
git switch -c feature-dashboard

Merging Branches

# Switch to main branch
git checkout main

# Merge a branch into current branch
git merge feature-login

# Delete a branch
git branch -d feature-login

# Force delete a branch
git branch -D feature-experimental

5. Working with GitHub

GitHub is a web-based hosting service for version control using Git.

Creating a Repository on GitHub

  1. Sign up for a GitHub account at github.com
  2. Click the "New" button to create a repository
  3. Give your repository a name and description
  4. Choose to make it public or private
  5. Initialize with a README if desired

Connecting Local Repository to GitHub

# Add remote origin
git remote add origin https://github.com/username/repository.git

# Push to GitHub
git push -u origin main

# Clone a repository from GitHub
git clone https://github.com/username/repository.git

Common GitHub Workflows

# Pull latest changes
git pull origin main

# Push your changes
git push origin main

# Create a new branch for a feature
git checkout -b feature-new-button

# After making changes
git add .
git commit -m "Add new button component"
git push origin feature-new-button

# Create a pull request on GitHub website

6. Version Control Best Practices

Following best practices helps maintain clean, organized, and collaborative codebases.

Commit Messages

  • Use present tense ("Add feature" not "Added feature")
  • Be concise but descriptive
  • Limit first line to 50 characters
  • Use blank line after first line
  • Explain "what" and "why", not "how"

Branching Strategies

# Git Flow (popular branching model)
main          # Production-ready code
develop       # Integration branch for features
feature/*     # Feature development branches
release/*     # Release preparation branches
hotfix/*      # Emergency fixes for production

Repository Organization

  • Maintain a clear README.md
  • Use .gitignore to exclude unnecessary files
  • Keep commits focused on single changes
  • Write meaningful commit messages
  • Use tags for releases

7. Resolving Conflicts

Conflicts occur when Git cannot automatically merge changes from different branches.

When Conflicts Happen

  • Two people modify the same line of code
  • One person deletes a file while another modifies it
  • Changes to the same part of a file in different branches

Identifying Conflicts

# When merging and conflict occurs
git merge feature-branch
# Auto-merging file.txt
# CONFLICT (content): Merge conflict in file.txt
# Automatic merge failed; fix conflicts and then commit the result.

Resolving Conflicts
