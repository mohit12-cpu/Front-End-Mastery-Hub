1. Introduction to React

React is a popular JavaScript library for building user interfaces. It allows developers to create reusable UI components and manage the state of their applications efficiently.

What is React?

React was developed by Facebook and released in 2013. It's used by many large companies including Instagram, Netflix, and Airbnb.

  • Component-based architecture
  • Virtual DOM for efficient rendering
  • One-way data flow
  • JSX syntax for writing HTML in JavaScript

Why Use React?

// Traditional JavaScript approach
document.getElementById('button').addEventListener('click', function() {
    document.getElementById('message').innerHTML = 'Hello, World!';
});

// React approach
function Button() {
    const [message, setMessage] = useState('Hello, World!');
    
    return (
        <button onClick={() => setMessage('Button clicked!')}>
            {message}
        </button>
    );
}

2. Components and Props

Components are the building blocks of React applications. Props (short for properties) are how components communicate with each other.

Functional Components

<Welcome name="Bob" />

Component Composition

3. State and Events

State is how React components manage data that can change over time. Events allow components to respond to user interactions.

Using State

function Counter() {
    const [count, setCount] = useState(0);
    
    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
        </div>
    );
}

Handling Events

function Form() {
    const [name, setName] = useState('');
    
    const handleSubmit = (e) => {
        e.preventDefault();
        alert(`Hello, ${name}!`);
    };
    
    return (
        <form onSubmit={handleSubmit}>
            <input 
                type="text" 
                value={name} 
                onChange={(e) => setName(e.target.value)} 
                placeholder="Enter your name"
            />
            <button type="submit">Submit</button>
        </form>
    );
}

4. Conditional Rendering

Conditional rendering allows you to display different components or elements based on certain conditions.

Using if Statements

function Greeting(props) {
    const isLoggedIn = props.isLoggedIn;
    
    if (isLoggedIn) {
        return <h1>Welcome back!</h1>;
    }
    return <h1>Please sign up.</h1>;
}

Using Logical && Operator

function Mailbox(props) {
    const unreadMessages = props.unreadMessages;
    
    return (
        <div>
            <h1>Hello!</h1>
            {unreadMessages.length > 0 && 
                <h2>
                    You have {unreadMessages.length} unread messages.
                </h2>
            }
        </div>
    );
}

5. Lists and Keys

React allows you to transform arrays into lists of elements using the map() function.

Rendering Lists

function NumberList(props) {
    const numbers = props.numbers;
    const listItems = numbers.map((number) =>
        <li key={number.toString()}>
            {number}
        </li>
    );
    
    return (
        <ul>{listItems}</ul>
    );
}

Importance of Keys

function TodoList(props) {
    const todos = props.todos;
    
    return (
        <ul>
            {todos.map((todo) => 
                <li key={todo.id}>
                    {todo.text}
                </li>
            )}
        </ul>
    );
}

6. JSX Examples

JSX is a syntax extension for JavaScript that looks similar to HTML. It allows you to write HTML-like code in your JavaScript files.

JSX Syntax

const element = <h1>Hello, world!</h1>;

const elementWithExpression = <h1>Hello, {name}!</h1>;

const elementWithAttributes = <div className="container">Content</div>;

Embedding Expressions

function formatName(user) {
    return user.firstName + ' ' + user.lastName;
}

const user = {
    firstName: 'Harper',
    lastName: 'Perez'
};

const element = (
    <h1>

7. Virtual DOM

The Virtual DOM is a programming concept where a virtual representation of the UI is kept in memory and synced with the real DOM by a library such as ReactDOM.

How Virtual DOM Works

1. When the state of an object changes, React creates a new Virtual DOM tree.

2. React compares the new Virtual DOM tree with the previous one (diffing).

3. React calculates the most efficient way to update the real DOM (reconciliation).

4. Only the changed elements are updated in the real DOM.

Benefits of Virtual DOM

  • Performance: Minimizes direct DOM manipulations
  • Efficiency: Only updates what's necessary
  • Abstraction: Simplifies UI development
  • Cross-platform: Enables React Native and other platforms
